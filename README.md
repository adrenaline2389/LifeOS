# LifeOS

LifeOS 是一个本地优先的人生操作系统雏形。

它不是云端效率工具，也不是心理诊断应用。LifeOS 的长期主体是六个本地子系统：个人生态、能量管理、认知管理、人生目标、人际关系和财务管理。它们共同帮助你在自己的设备上管理身体、心理、信息、行动、关系和资源。

当前开发版本是 `v1.4`：财务管理系统的基础设施版本。它继承 v1.2 的个人生态系统闭环和 v1.3 的能量管理系统闭环，并把“财务管理系统”从后续开放入口推进为可进入的本地子系统。

## 核心原则

- **本地优先**：个人生活数据默认只保存在你的设备上。
- **用户拥有数据**：当前版本先保留本地保存和重置，导入导出会在未来个人说明书板块成熟后重新设计。
- **不需要账号**：当前版本没有登录、注册、云同步或多人协作。
- **不做诊断**：LifeOS 不提供心理诊断、人格判断、医疗建议或健康建议。
- **不使用 AI 分析用户**：当前版本通过确定性规则和用户回答生成启动扫描结果，并通过本地确定性规则生成生态观察摘要、能量风向标和钱包余额汇总。
- **所有观察都待验证**：系统生成的内容只是线索，不是最终结论。

## LifeOS 的主体

LifeOS 的长期结构不是把人生拆成一堆互不相关的分类，而是把一个人的运行方式拆成几个互相影响的系统。

### 个人生态系统

个人生态系统负责你的生理基础和生活环境。

它关心：

- 作息
- 睡眠
- 饮食
- 饮水
- 身体状态
- 运动
- 居住环境
- 日常维护
- 生活节律

它回答的问题是：

> 我的身体和环境能不能支撑我运行？

### 能量管理系统

能量管理系统负责你的心理余量和恢复方式。

它关心：

- 心力
- 情绪负载
- 压力信号
- 低能量状态
- 恢复方式
- 过载预警
- 社交耐心
- 心理缓冲区

它回答的问题是：

> 我现在有没有心理余量？

个人生态系统更偏体力，能量管理系统更偏心力。两者会互相影响，但不应该混成一个系统。

### 认知管理系统

认知管理系统负责信息、学习、判断和反思。

它关心：

- 想法记录
- 学习材料
- 认知模式
- 判断依据
- 复盘
- 长期知识沉淀

它回答的问题是：

> 我如何理解世界、整理信息，并更新自己的判断？

### 人生目标管理系统

人生目标管理系统负责方向、项目和行动。

它关心：

- 长期方向
- 当前目标
- 项目推进
- 下一步行动
- 节奏选择
- 阻力识别

它回答的问题是：

> 我想往哪里走，下一步该怎么动？

### 人际关系管理系统

人际关系管理系统负责连接、沟通和边界。

它关心：

- 沟通偏好
- 关系说明
- 协作方式
- 边界
- 雷区
- 支持网络
- 重要关系记录

它回答的问题是：

> 我如何和别人连接、协作，并保护自己的边界？

### 财务管理系统

财务管理系统负责资源、消费和自由度。

它关心：

- 收入
- 支出
- 消费模式
- 财务目标
- 风险感
- 选择空间
- 长期自由度

它回答的问题是：

> 我的资源如何流动，它们如何影响我的选择？

v1.4 只开启财务管理系统的基础设施。真正可用的财务业务只有“我的钱包”：用户手动维护资金容器和当前余额，系统只做本地汇总与展示，不提供预算工具、投资建议、财务诊断、银行连接或支付连接。

## 当前版本能做什么

### 1. 首次启动扫描

第一次打开 LifeOS 时，你会进入一个轻量 onboarding 流程。

它包含 9 道问题，围绕这些主题收集初始信号：

- 状态下降时最先失灵的部分
- 恢复方式
- 推进事情的节奏
- 沟通偏好
- 低状态下的雷区
- 长期相处或合作时别人应知道的事
- 最近想改善的方向
- 希望 LifeOS 先帮助你成为什么样的人
- 想留给未来自己的提醒

第 1-8 题支持多选和“其他”文本回答，第 9 题可以跳过。

这些问题的作用是做一次初始扫描，帮助你从六个子系统中判断可以先开启哪一个。它不是 LifeOS 的核心产品，也不应该承载过多长期功能。

### 2. LifeOS 启动面板

完成 onboarding 后，LifeOS 会生成一个启动面板。

启动面板包括：

- 初始扫描状态
- 建议优先开启的子系统
- 六个子系统地图
- 带来源回答的扫描线索
- 本地数据重置入口

启动面板不是完整个人说明书，也不是最终主界面。它是把用户带入六个子系统的入口。

### 3. 子系统建议

LifeOS 会根据 onboarding 回答，最多推荐 2 个适合优先开启的子系统。

v1.4 可以推荐：

- 个人生态系统
- 能量管理系统
- 认知管理系统
- 人生目标管理系统
- 人际关系管理系统
- 财务管理系统

推荐结果会保留来源回答，方便你知道它为什么出现。

当前可进入的真实子系统是个人生态系统、能量管理系统和财务管理系统。认知管理系统、人生目标管理系统和人际关系管理系统仍显示后续开放反馈。

### 4. 个人生态系统

v1.2 开启第一个真实子系统：个人生态系统。v1.3 继续保留这个闭环。

它不是健康管理、习惯打卡或诊断系统。它只帮助你在本地观察身体和生活环境是否支撑自己运行。

个人生态系统包含 6 个维度：

- 睡眠恢复
- 作息节律
- 身体状态
- 饮食饮水
- 活动舒展
- 环境支撑

你可以选择性记录某一个维度，不需要每天填写整张表。每次记录会形成一个本地生态观察点。

个人生态系统包含：

- 当前生态状态
- 今日生态轨迹
- 生活晴雨表

生活晴雨表支持查看过去 1 天、7 天、15 天、30 天的观察摘要。界面展示语义状态，不展示内部分数，不提供连续打卡、完成率或惩罚文案。

### 5. 能量管理系统

v1.3 开启第二个真实子系统：能量管理系统。

它不是心理诊断、情绪分析、健康建议、任务规划或恢复建议系统。它只帮助你在本地观察自己的心理余量与恢复状态。

能量管理系统包含 6 个维度：

- 当前余量
- 压力负载
- 情绪天气
- 注意带宽
- 社交电量
- 行动阻力

你可以选择性记录某一个维度，不需要每天填写整张表。每次记录会形成一个本地能量观察点。

能量管理系统包含：

- 当前能量状态
- 今日能量轨迹
- 能量风向标

能量风向标支持查看过去 1 天、7 天、15 天、30 天的观察摘要。界面展示语义状态，不展示内部分数，不提供连续打卡、完成率或惩罚文案。

### 6. 财务管理系统

v1.4 开启第三个真实子系统：财务管理系统。

它不是记账软件、预算惩罚工具、理财建议系统、投资建议系统或财务诊断系统。它只先帮助你在本地维护一份手动的余额快照。

财务管理系统当前真正可用的业务只有：

- 我的钱包

“我的钱包”支持：

- 新建资金容器
- 直接编辑容器当前余额
- 为容器选择颜色
- 填写可选备注
- 删除资金容器
- 自动汇总当前总余额
- 总余额为正时展示彩色环形分布和容器占比
- 总余额为零时展示灰色状态，不展示占比
- 总余额为负时展示灰色状态，不展示占比，并显示当前负债金额

资金容器可以是银行卡、现金、钱包、基金账户或用户自己定义的任何本地余额容器。系统不要求分类，不连接真实银行或支付平台。

财务管理系统页面会展示金鹅账户、梦想账户、金钱流入来源、日常开销池和财富流动日志的后续开放占位，但 v1.4 不提供这些业务的编辑或保存能力。

### 7. 本地保存与重置

当前版本会把 onboarding 回答、启动扫描结果、个人生态系统观察点、能量管理系统观察点和钱包容器保存在浏览器本地 IndexedDB 中。

你可以在明确确认后清空本地 LifeOS 数据。重置会清空当前浏览器中的 LifeOS 本地数据，并回到首次启动状态。

v1.4 不提供用户侧 JSON 导出、Markdown 导出或完整个人说明书编辑入口。

## 后期板块

### 个人说明书

个人说明书是未来版本中的独立板块，很像一个由六个子系统沉淀出来的个人备份文件，但它不是第七个系统。

长期来看，它会根据你在六个子系统中的使用情况，总结出一份可读、可迁移的 Markdown 说明书。到这个阶段，导入和导出功能也应围绕这份说明书来设计：导出时得到一份 `.md` 文件，导入时上传这份文件，LifeOS 可以像恢复备份一样重建六个系统的关键状态。

v1.4 不开发这个板块，只为后续版本保留方向。

## 系统之间的关系

可以把 LifeOS 理解成这样：

```text
LifeOS 主体 = 六个子系统

个人生态系统 = 生理基础与生活环境
能量管理系统 = 心理余量与恢复
认知管理系统 = 信息、学习、判断、反思
人生目标管理系统 = 方向、项目、行动
人际关系管理系统 = 连接、沟通、边界
财务管理系统 = 资源、消费、自由度

辅助层:
首次启动引导 = 初始扫描 / 推荐器
个人说明书 = 后期沉淀 / Markdown 备份文件
```

其中，个人生态系统和能量管理系统是一对底层系统：

- 个人生态系统看身体和环境是否能支撑你。
- 能量管理系统看心理和情绪是否能支撑你。

认知管理系统帮助你看清现实，人生目标管理系统把方向转成行动，人际关系管理系统处理你和其他人的连接，财务管理系统处理资源约束和自由度。v1.4 中，财务管理系统先从“我的钱包”开始，只建立本地余额快照基础设施。

## 运行方式

安装依赖：

```bash
npm install
```

启动本地开发服务：

```bash
npm run dev
```

然后在浏览器打开：

```text
http://localhost:3000
```

```bash
npm run dev

npm run typecheck

```

## 技术栈

- Next.js
- React
- TypeScript
- IndexedDB
- Dexie
- Vitest

LifeOS 当前把 Next.js 作为本地 Web App 框架使用。个人生活数据相关逻辑运行在浏览器端，不依赖服务端数据库。

- 账号系统
- 云同步
- 云端数据库
- 多人协作
- AI 分析
- AI 心理诊断
- 健康建议
- 医疗建议
- 连续打卡
- 显性数字评分
- JSON 导出
- Markdown 导出
- 完整个人说明书编辑器
- 其他三个子系统的真实业务闭环
- 完整财务记账
- 预算工具
- 梦想账户
- 金鹅账户
- 金钱流入来源
- 日常开销池
- 财富流动日志
- 投资建议
- 财务诊断
- 银行连接
- 支付连接
- 完整目标树
- 完整关系 CRM
