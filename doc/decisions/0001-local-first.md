# 0001: 本地优先

## 状态

Accepted

## 背景

LifeOS 管理的是个人生活、状态、偏好、关系、目标和自我说明书一类数据。这些数据比普通应用配置更敏感，也更容易在长期使用中积累出完整的个人画像。

v1.0 的目标不是先做一个云端效率工具，而是做一个用户可以在自己设备上运行、自己掌控数据的个人操作系统雏形。

## 决策

LifeOS 采用本地优先原则。

v1.0 中：

- 用户数据保存在用户自己的设备上。
- 默认不向任何云端服务上传个人生活数据。
- 不实现账号系统。
- 不实现云同步。
- 不使用云端数据库保存个人生活数据。
- 不通过 AI 服务分析、解释或诊断用户。
- 导出能力以 JSON 和 Markdown 为主，让用户可以自行备份和迁移。

本地持久化使用 IndexedDB，并通过 Dexie 封装。

## 原因

本地优先能让 LifeOS 的初始版本更符合项目定位：

- 用户拥有自己的数据。
- 项目可以先作为个人本地应用跑通，不依赖云端账号和部署。
- 隐私边界清晰，减少早期架构复杂度。
- 开源用户可以 clone 后运行自己的实例。
- 未来即使增加同步能力，也可以建立在明确的数据所有权原则上。

## 后果

正面影响：

- v1.0 可以避免账号、权限、后端、安全审计和云端成本。
- 用户个人数据不会因为默认架构而离开本机。
- 数据导出和重置成为清晰的一等能力。

代价：

- v1.0 不支持跨设备同步。
- v1.0 不支持多人协作。
- 浏览器本地数据可能受用户清理浏览器数据、换设备或浏览器限制影响。
- 后续如果增加同步，需要明确设计加密、授权、冲突处理和退出机制。

## 约束

任何未来版本如果引入云同步、账号系统、远程备份或 AI 分析，都必须作为新的显式决策记录，而不能默认为本地优先原则的自然延伸。

默认原则仍然是：

> 用户拥有自己的 LifeOS 数据，云端能力只能作为可选择的附加层。
