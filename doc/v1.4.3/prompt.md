# LifeOS v1.4.3 Agent Prompt

## 目标

根据当前项目文档，实现 LifeOS v1.4.3。

v1.4.3 是财务管理系统的第三个增量业务版本。它建立在 v1.4 已完成的“我的钱包”本地余额快照闭环、v1.4.1 已完成的“收入来源”手动入账闭环和 v1.4.2 已完成的“日常开销池”手动划款与消费扣减闭环之上，新增第四个真实财务面板：

> 财富流动日志。

v1.4.3 的新增闭环是：

```text
进入财务管理系统
  -> 看到“财富流动日志”
  -> 收入来源手动入账成功
  -> 生成入账日志
  -> 日常开销池划款成功
  -> 生成划款日志
  -> 日常开销池消费成功
  -> 生成消费日志
  -> 删除消费流水并回滚开销池余额
  -> 生成消费回退日志
```

实现过程应支持多 agent 协作。主 Agent 负责整体集成、任务分派、进度跟踪和最终验证；子 Agent 按 `doc/v1.4.3/tasks/` 中的任务文件领取模块。

## 数据模型要求

财富流动事件类型建议模型：

```ts
type WealthFlowEventType =
  | "income_received"
  | "daily_expense_transfer"
  | "daily_expense_spent"
  | "daily_expense_refund";
```

财富流动方向建议模型：

```ts
type WealthFlowDirection = "in" | "out" | "transfer";
```

财富流动主体建议模型：

```ts
type WealthFlowSubjectType =
  | "wallet_container"
  | "income_source"
  | "daily_expense_pool"
  | "daily_expense_entry"
  | "dream_account"
  | "golden_goose_account";

type WealthFlowSubjectSnapshot = {
  type: WealthFlowSubjectType;
  id?: string;
  nameSnapshot: string;
};
```

财富流动日志建议模型：

```ts
type WealthFlowEvent = {
  id: string;
  type: WealthFlowEventType;
  direction: WealthFlowDirection;
  amount: number;
  occurredAt: string;
  source?: WealthFlowSubjectSnapshot;
  target?: WealthFlowSubjectSnapshot;
  relatedEventId?: string;
  relatedDailyExpenseEntryId?: string;
  note?: string;
  createdAt: string;
  updatedAt: string;
};
```

实现时可以按现有类型风格调整命名，但必须保留这些概念：

- 日志 ID。
- 事件类型。
- 事件方向。
- 正数金额。
- 发生时间。
- 来源快照。
- 目标快照。
- 关联原事件 ID。
- 关联消费流水 ID。
- 可选备注。
- 创建时间。
- 更新时间。

不要新增：

- `Transaction`。
- `LedgerEntry`。
- `BudgetCategory`。
- `RefundEntry` 通用模型。
- `WalletTransfer` 任意转账模型。
- `DreamAccount` 业务模型。
- `GoldenGooseAccount` 业务模型。

## 入账日志规则

```text
用户对收入来源执行手动入账
  -> 校验收入来源和目标钱包容器可用
  -> 计算本次入账金额
  -> 更新目标钱包容器余额
  -> 创建 WealthFlowEvent(type = income_received)
  -> 保存钱包容器
  -> 保存财富流动日志
  -> 刷新我的钱包和财富流动日志
```

如果入账失败，不创建财富流动日志。

## 划款日志规则

```text
用户从钱包容器划入日常开销池
  -> 校验来源钱包容器可用
  -> 校验划款金额
  -> 更新来源钱包容器余额
  -> 更新日常开销池余额和最近一次划款信息
  -> 创建 WealthFlowEvent(type = daily_expense_transfer)
  -> 保存钱包容器
  -> 保存日常开销池
  -> 保存财富流动日志
  -> 刷新我的钱包、日常开销池和财富流动日志
```

如果划款失败，不创建财富流动日志。

## 消费日志规则

```text
用户在日常开销池中消费扣款
  -> 校验消费金额和备注
  -> 更新日常开销池余额
  -> 创建 DailyExpenseEntry
  -> 创建 WealthFlowEvent(type = daily_expense_spent)
  -> 保存日常开销池
  -> 保存消费流水
  -> 保存财富流动日志
  -> 刷新日常开销池和财富流动日志
```

如果消费失败，不创建财富流动日志。

## 消费回退日志规则

```text
用户删除某条消费流水
  -> 找到 DailyExpenseEntry
  -> 找到可关联的原消费日志
  -> 更新日常开销池余额
  -> 删除 DailyExpenseEntry
  -> 创建 WealthFlowEvent(type = daily_expense_refund)
  -> 保存日常开销池
  -> 删除消费流水
  -> 保存财富流动日志
  -> 刷新日常开销池和财富流动日志
```

消费回退不修改钱包容器余额。

消费回退不删除原消费日志。

## UI 要求

财富流动日志面板应展示：

- 面板标题。
- 本地记录说明。
- 空状态。
- 事件列表。
- 事件类型。
- 金额方向。
- 发生时间。
- 来源和目标快照。
- 备注或说明。
- 消费回退关联提示。

展示方向：

- 入账展示为 `+金额`。
- 划款展示为“划入开销池 金额”。
- 消费展示为 `-金额`。
- 消费回退展示为 `+金额`。

空状态文案应表达：

```text
还没有从当前版本开始记录的财富流动。
```

不要提示用户补录旧数据。

## 多 Agent 协作方式

建议按以下顺序执行：

1. 财富流动日志类型与本地存储。
2. 财富流动日志业务记录动作。
3. 财富流动日志面板。
4. 日常开销池用户侧收敛。
5. 财务管理系统页面集成。
6. 全量回归与验收。

每个子 Agent 完成自己的任务后，应：

- 更新对应任务文件 checklist。
- 更新 `doc/v1.4.3/tasks/progress.md`。
- 说明运行过的验证命令。
- 不修改无关历史版本文档。
