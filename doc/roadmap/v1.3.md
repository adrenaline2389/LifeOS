# LifeOS v1.3 Roadmap

## 版本定位

LifeOS v1.3 计划开启第二个真实子系统闭环：能量管理系统。

v1.2 已经完成个人生态系统的最小观察闭环。v1.3 不修改 v1.2 的版本边界，也不把新需求写回 `doc/v1.2/`。v1.3 在 v1.2 的模式上继续前进，把能量管理系统从“后续开放”推进为可使用的真实子系统。

v1.3 的目标是：

> 建立能量管理系统的最小观察闭环。

能量管理系统在 v1.3 中不是心理诊断、情绪分析、效率管理、任务规划或恢复建议系统。它是一个本地的心理余量雷达，帮助用户记录和回顾自己当下的心理余量、压力负载、情绪天气、注意带宽、社交电量和行动阻力。

相关架构决策：

- `doc/decisions/0001-local-first.md`
- `doc/decisions/0002-nextjs-as-local-web-app.md`
- `doc/decisions/0003-six-subsystems-as-core.md`
- `doc/decisions/0004-semantic-observation-events.md`

## 继承自 v1.2 的边界

v1.3 继续继承：

- 本地优先。
- 用户数据保存在用户自己的设备上。
- 不使用账号系统。
- 不使用云同步。
- 不使用云端数据库保存个人生活数据。
- 不使用 AI 分析、AI 推理、心理诊断、人格诊断、情绪诊断或医疗建议。
- 技术栈继续使用 Next.js + React + TypeScript。
- 本地持久化继续使用 IndexedDB + Dexie。
- 界面语言使用简体中文。
- 保留复古 Macintosh 风格和旧电脑感。
- onboarding 仍然只是初始扫描和子系统推荐器。
- 个人说明书仍然是后期板块，不是当前主功能。
- 子系统回顾必须是本地确定性逻辑，不得伪装成诊断结论。

v1.3 不修改 `doc/v1.2/` 的历史边界。v1.2 仍然是个人生态系统最小观察闭环版本；v1.3 的新需求进入 roadmap，后续确认后再进入独立版本目录。

## 与个人生态系统的关系

个人生态系统管理生理基础与生活环境。它观察的是身体和环境是否支撑用户运行。

能量管理系统管理心理余量与恢复。v1.3 第一版只观察“当下还有多少心理/行动余量，以及哪些状态正在消耗或阻滞用户”。

v1.3 不做跨系统推断：

- 不根据睡眠、饮食、活动等生态观察自动判断能量状态。
- 不根据能量观察反推身体、健康或心理状况。
- 不把两个子系统的数据合并成评分、诊断或建议。
- 不因为能量低而生成“你应该做什么”的提示。

个人生态系统和能量管理系统可以共享语义观察点模型，但实现时应保持模块边界清晰，避免为了复用而破坏 v1.2 已完成闭环。

## 核心闭环

v1.3 的核心使用路径：

1. 用户从 LifeOS 启动面板进入能量管理系统。
2. 用户看到当前能量状态。
3. 用户可以选择性更新某一个能量维度，而不是被要求填写整张表。
4. 每次更新生成一个本地“能量观察点”。
5. 用户查看今日能量轨迹。
6. 用户查看过去 1 天、7 天、15 天、30 天的能量风向标。

最小闭环是：

```text
进入能量管理系统
  -> 看到当前能量状态
  -> 选择性记录一个能量观察点
  -> 本地保存
  -> 查看今日能量轨迹
  -> 查看能量风向标
```

用户可以只查看，不记录。系统不能在进入页面时强制弹出记录表，也不能把记录做成连续打卡压力。

同一次进入能量管理系统页面时，同一个维度最多保留一个新观察点。用户在同一页面会话里再次选择同一维度时，系统应覆盖刚刚记录的那一条，而不是不断追加挤满今日轨迹。用户返回启动面板并重新进入后，才可以为同一维度产生新的观察点。

## 产品结构

### 面板 A: 当前能量状态

当前能量状态是能量管理系统的主面板。

它展示六个维度的最近状态：

1. 当前余量
2. 压力负载
3. 情绪天气
4. 注意带宽
5. 社交电量
6. 行动阻力

每个维度是一张状态卡，包含：

- 维度名称。
- 一句说明。
- 当前或最近一次语义状态。
- 记录时间。
- 一个轻量的更新入口。

如果某个维度今天没有记录，应显示“今日未观察”。如果某个维度从未记录，应显示“尚未观察”。如果最近一次记录不是今天，可以显示最近记录日期，但必须避免暗示它代表今天。

### 面板 B: 今日能量轨迹

今日能量轨迹展示当天 0 点以后产生的能量观察点。

要求：

- 按维度分组。
- 每个观察点显示语义状态和时刻。
- 每个观察点提供删除入口，用于处理误触。
- 没有记录的时刻保持空白。
- 不对空白时段做插值。
- 不用折线连接不连续的点，避免暗示系统知道中间状态。

“0 点以后清零”只表示今日轨迹面板切换到新的一天，不表示删除历史数据。历史观察点仍用于能量风向标。

### 面板 C: 能量风向标

能量风向标负责回顾过去一段时间的能量状态。

支持范围：

- 1 天
- 7 天
- 15 天
- 30 天

每个范围内，按维度展示：

- 观察次数。
- 平均语义状态。
- 低位观察次数。
- 最近一次观察。
- 一句确定性摘要。

摘要必须使用“观察”“记录”“线索”“这段记录里”一类语言，不能使用“诊断”“建议”“你应该”“说明你已经”一类语言。

示例文案方向：

```text
过去 7 天，当前余量的记录多落在「够用」附近。
压力负载记录 6 次，其中 2 次落在「偏重」或更低。
行动阻力这段记录接近「有点卡」，这只是一条本地观察线索。
```

## 能量维度

v1.3 第一版包含六个维度。维度顺序固定为：

1. 当前余量
2. 压力负载
3. 情绪天气
4. 注意带宽
5. 社交电量
6. 行动阻力

维度必须配置化。新增维度应优先通过配置完成，而不是复制一套业务逻辑。

### 语义选项与内部状态

用户选择语义状态。系统内部可以把语义状态映射为 `internalScore`，用于确定性统计。

内部状态继续使用 7 档：

```text
+3 很支撑
+2 支撑
+1 略微支撑
 0 中性或可用
-1 略微拖累
-2 拖累
-3 明显拖累
```

`没观察` 不是 7 档之一，也不应写入平均值。没有观察就是没有数据。

### 当前余量

当前余量回答：我现在还有多少心理和行动电量。

| 内部状态 | 用户语义 |
| --- | --- |
| +3 | 很充足 |
| +2 | 有余量 |
| +1 | 够用 |
| 0 | 勉强可用 |
| -1 | 偏低 |
| -2 | 快见底 |
| -3 | 已透支 |

### 压力负载

压力负载回答：我现在背着的压力重不重。

对这个维度来说，负载越轻越支撑，负载越重越拖累。

| 内部状态 | 用户语义 |
| --- | --- |
| +3 | 很轻 |
| +2 | 可承载 |
| +1 | 稍有压力 |
| 0 | 有点满 |
| -1 | 偏重 |
| -2 | 很重 |
| -3 | 过载 |

### 情绪天气

情绪天气回答：我的内在气候如何。

这个维度使用天气感语言，不使用诊断或病理标签。

| 内部状态 | 用户语义 |
| --- | --- |
| +3 | 明亮 |
| +2 | 平稳 |
| +1 | 柔和 |
| 0 | 阴晴不定 |
| -1 | 敏感 |
| -2 | 低沉 |
| -3 | 麻木/失控 |

### 注意带宽

注意带宽回答：我现在的脑内空间和处理能力够不够。

这个维度只观察当下能量状态，不提前实现认知管理系统。

| 内部状态 | 用户语义 |
| --- | --- |
| +3 | 清晰宽裕 |
| +2 | 能聚焦 |
| +1 | 基本在线 |
| 0 | 有点散 |
| -1 | 容易分心 |
| -2 | 很难处理 |
| -3 | 大脑宕机 |

### 社交电量

社交电量回答：我现在能不能和人互动，需要多少边界。

这个维度只观察当下互动余量，不提前实现人际关系管理系统。

| 内部状态 | 用户语义 |
| --- | --- |
| +3 | 想连接 |
| +2 | 可以交流 |
| +1 | 少量可用 |
| 0 | 看情况 |
| -1 | 需要边界 |
| -2 | 不想说话 |
| -3 | 完全封闭 |

### 行动阻力

行动阻力回答：我想行动时顺不顺，有没有启动困难或抗拒。

对这个维度来说，阻力越低越支撑，阻力越高越拖累。这个维度只观察启动摩擦，不提前实现人生目标管理系统。

| 内部状态 | 用户语义 |
| --- | --- |
| +3 | 很顺手 |
| +2 | 可以启动 |
| +1 | 慢慢能动 |
| 0 | 有点卡 |
| -1 | 难启动 |
| -2 | 抗拒 |
| -3 | 停摆 |

## 数据模型方向

v1.3 应新增能量管理系统的本地数据模型。

实现时可以沿用 v1.2 的语义观察点模型，但建议在 v1.3 中保持能量观察点的模块独立，避免为复用而改动个人生态系统的已完成数据表和闭环。

建议概念模型：

```ts
type EnergyDimensionId =
  | "currentCapacity"
  | "pressureLoad"
  | "emotionalWeather"
  | "attentionBandwidth"
  | "socialBattery"
  | "actionResistance";

type EnergyInternalScore = -3 | -2 | -1 | 0 | 1 | 2 | 3;

type EnergySemanticValue = {
  id: string;
  label: string;
  internalScore: EnergyInternalScore;
};

type EnergyDimensionDefinition = {
  id: EnergyDimensionId;
  label: string;
  description: string;
  values: EnergySemanticValue[];
};

type EnergyObservation = {
  id: string;
  dimensionId: EnergyDimensionId;
  valueId: string;
  valueLabel: string;
  internalScore: EnergyInternalScore;
  observedAt: string;
  note?: string;
};
```

观察点必须保存 `valueLabel` 和 `internalScore` 快照，避免未来调整维度配置后重新解释历史记录。

本地持久化方向：

- 新增能量观察点保存能力。
- 新增能量观察点删除能力。
- 新增按时间范围读取能量观察点能力。
- 重置本地数据时，应同时清空 onboarding、启动扫描结果、生态观察点和能量观察点。
- 不把能量观察点上传到网络、服务端、云数据库或 AI 服务。

## 本地时间规则

今日能量轨迹按浏览器本地日期分组。

原则：

- 今日从用户设备本地时间 00:00 开始。
- 1 天、7 天、15 天、30 天统计同样使用本地日期边界。
- 0 点后今日视图自然为空，但历史数据不删除。
- 不引入服务器时间。

## 聚合规则方向

v1.3 的聚合必须是本地确定性的，不使用 AI。

### 当前能量状态

每个维度取最近一次观察点。

展示时区分：

- 今日已观察。
- 最近观察不是今天。
- 从未观察。

### 今日能量轨迹

筛选 `observedAt` 位于本地今天范围内的观察点。

按维度分组，再按时间升序展示。

### 能量风向标

对指定范围内的观察点按维度分组。

每个维度计算：

- `observationCount`
- `averageInternalScore`
- `lowObservationCount`
- `latestObservation`
- `summaryLabel`

`lowObservationCount` 统计 `internalScore <= -1` 的观察点。

`summaryLabel` 由平均值映射回该维度最接近的语义状态。它不是诊断，只是这段记录的平均位置。

v1.3 不包含：

- 心理诊断。
- 情绪诊断。
- 人格诊断。
- 医疗建议。
- 健康建议。
- AI 分析。
- AI 推理。
- 账号系统。
- 云同步。
- 云端数据库。
- 服务器端保存个人生活数据。
- 用户直接编辑内部分数。
- 1 到 5 分或 1 到 10 分的显性评分 UI。
- 连续打卡、连续天数、惩罚或完成率压力。
- 任务规划。
- 日程安排。
- 今日能量预算。
- 恢复方案推荐。
- 恢复方式库。

- 跨系统推断或合并评分。
- 认知管理系统真实业务闭环。
- 人生目标管理系统真实业务闭环。
- 人际关系管理系统真实业务闭环。
- 财务管理系统真实业务闭环。
- 完整个人说明书。
- JSON 导出。
- Markdown 导出。
- Markdown 导入。
- 备份恢复。

## 与启动面板的关系

v1.1 启动面板中六个子系统入口已经存在。v1.2 已将个人生态系统入口推进为可进入状态。

v1.3 应将能量管理系统入口从“后续开放”推进为可进入状态。

要求：

- 点击能量管理系统进入能量管理系统页面。
- 能量管理系统被推荐时，推荐卡片也能进入同一页面。
- 个人生态系统仍然可进入。
- 其他四个子系统仍显示后续开放反馈。
- 从能量管理系统页面可以返回 LifeOS 启动面板。

v1.3 不应为了导航完整性而提前实现认知管理系统、人生目标管理系统、人际关系管理系统或财务管理系统。

## 验收标准方向

v1.3 完成时应满足：

- 新用户仍然可以完成首次启动扫描。
- 完成扫描后进入 LifeOS 启动面板。
- 启动面板展示六个子系统。
- 个人生态系统仍然可以进入并保持 v1.2 闭环。
- 能量管理系统可以进入。
- 其他四个子系统仍显示后续开放反馈。
- 能量管理系统展示六个维度。
- 用户可以只记录一个能量维度。
- 更新控件用七档语义轴展示状态程度。
- 同一次进入页面内，同一维度重复记录会覆盖本次进入刚生成的观察点。
- 用户可以删除误触生成的能量观察点。
- 记录后当前能量状态更新。
- 今日能量轨迹展示当天观察点。
- 能量风向标支持 1 天、7 天、15 天、30 天。
- UI 不展示内部分数。
- UI 不强制用户记录。
- UI 没有连续打卡、完成率或惩罚文案。
- UI 没有心理诊断、情绪诊断、医疗建议、健康建议、AI 分析或任务规划文案。
- 重置后清空 onboarding、启动扫描、生态观察点和能量观察点。
- 用户界面没有 JSON 导出入口。
- 用户界面没有 Markdown 导出入口。
- 用户界面没有完整个人说明书编辑入口。
- 不引入账号、云同步、云数据库或 AI 分析。
