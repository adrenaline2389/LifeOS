# LifeOS v1.1 Roadmap

## 版本定位

v1.1 是一次减法重构版本。

它的目标不是在 v1.0 上继续堆功能，而是把 LifeOS 从“个人说明书原型”纠偏为“六个子系统驱动的本地人生操作系统启动器”。

v1.1 完成后，用户完成首次启动扫描后看到的重点不再是个人说明书，而是六个子系统的地图、状态和推荐开启顺序。

## 核心目标

1. 删除 v1.0 中过早出现的个人说明书编辑和导出能力。
2. 将 9 道 onboarding 题重新定位为“初始扫描”和“子系统推荐器”。
3. 新增“个人生态系统”作为第六个顶层子系统。
4. 将完成 onboarding 后的主界面改为 LifeOS 启动面板。
5. 让 v1.1 的主体验围绕六个子系统入口和推荐理由展开。

## 非目标

v1.1 不做：

- 完整个人说明书。
- 个人说明书编辑器。
- JSON 导出。
- Markdown 导出。
- Markdown 导入。
- 备份恢复。
- 真实子系统业务数据。
- 账号系统。
- 云同步。
- 云端数据库。
- AI 分析、AI 推理或心理诊断。

## 产品结构

### 主体：六个子系统

v1.1 的主界面应展示六个子系统：

1. 个人生态系统：生理基础与生活环境。
2. 能量管理系统：心理余量与恢复。
3. 认知管理系统：信息、学习、判断和反思。
4. 人生目标管理系统：方向、项目和行动。
5. 人际关系管理系统：连接、沟通和边界。
6. 财务管理系统：资源、消费和自由度。

每个子系统在 v1.1 中都可以是入口卡片，不需要实现真实业务闭环。

### 辅助：首次启动扫描

9 道 onboarding 题保留，但文案和数据目标应从“生成个人说明书”改为“识别优先开启的系统”。

完成扫描后，系统应输出：

- 初始扫描完成状态。
- 六个子系统地图。
- 最多 2 个建议优先开启的子系统。
- 每个推荐的来源回答。
- 少量“扫描线索”，作为推荐依据，而不是自我结论。

### 后期：个人说明书

个人说明书在 v1.1 中不作为主功能出现。

它应被定位为未来独立板块：由六个子系统的长期使用情况沉淀生成，并在后续版本中承担 Markdown 导入、导出和恢复备份的角色。

## 功能删减

### 删除用户侧 JSON 导出

v1.1 不再在用户界面提供 JSON 导出入口。

可以删除相关 UI，也可以保留底层测试工具函数，但不能作为产品能力展示给用户。

### 删除用户侧 Markdown 导出

v1.1 不再在用户界面提供 Markdown 导出入口。

Markdown 导出会在未来个人说明书成熟后重新设计。

### 删除完整个人说明书编辑入口

v1.1 不再提供完整个人说明书编辑界面。

已有的说明书章节生成和编辑逻辑可以被删除、闲置或降级为内部数据，但不应出现在主体验中。

### 删除自我清晰度主概念

“自我清晰度：朦胧”不再作为主界面核心区块展示。

可以替换为：

```text
启动状态：初始扫描完成
```

## Onboarding 修正方向

9 道题仍然保留，但需要修正题目目标、signal tags 和 write targets。

### 题目目标

题目应服务于六个子系统推荐，而不是生成说明书。

### 个人生态系统信号

新增 `ecosystem` 子系统信号。

适合映射到个人生态系统的信号包括：

- 睡眠
- 饮食
- 饮水
- 运动
- 整理环境
- 生活秩序
- 作息与精力
- 身体不适
- 日常维护
- 居住环境

### 推荐规则

推荐结果最多 2 个。

推荐依据应来自用户回答中的子系统信号。推荐理由应解释“为什么建议先看这个系统”，而不是给用户下判断。

### 个人说明书相关信号

`manual` 不再作为可推荐子系统。

与“尚不清楚”“我还没观察清楚”“给未来自己的备注”相关的回答，可以作为扫描线索保留，但不应推荐“个人说明书系统”。

## v1.1 完成后的用户流程

1. 用户打开 LifeOS。
2. 如果没有本地扫描记录，进入首次启动扫描。
3. 用户完成 9 道题。
4. LifeOS 保存扫描回答。
5. LifeOS 计算六个子系统的初始推荐。
6. 用户进入 LifeOS 启动面板。
7. 启动面板展示六个子系统入口。
8. 被推荐的 1-2 个子系统显示“建议优先开启”。
9. 用户可以查看推荐来源。
10. 用户可以重置本地数据，重新扫描。

## 数据模型方向

v1.1 可以继续沿用 v1.0 的本地存储能力，但命名应逐步从 `ManualProfile` 迁移到更符合当前方向的结构。

建议新增或替代为：

```ts
type SubsystemId =
  | "ecosystem"
  | "energy"
  | "cognition"
  | "goals"
  | "relationships"
  | "finance";

type StartupScanProfile = {
  version: "1.1";
  completedAt: string;
  scanStatus: "completed";
  scanClues: StartupScanClue[];
  suggestedSubsystems: SuggestedSubsystem[];
};

type StartupScanClue = {
  id: string;
  text: string;
  sourceAnswerRefs: SourceAnswerRef[];
};
```

是否保留旧 `ManualProfile` 作为迁移兼容，由实现阶段决定。v1.1 不需要提供正式数据迁移工具，但刷新后应能正常读取 v1.1 新扫描结果。

## 文案方向

旧文案：

```text
个人说明书控制面板
系统刚开始认识你
完整个人说明书
导出 JSON
导出 Markdown
```

新文案：

```text
LifeOS 启动面板
初始扫描完成
建议优先开启的系统
查看推荐来源
重置本地数据
```

## 验收标准

v1.1 完成时应满足：

- 页面主标题不再是“个人说明书控制面板”。
- 主界面展示六个子系统，而不是说明书章节。
- “个人生态系统”出现在六个子系统中。
- 推荐系统最多推荐 2 个子系统。
- 推荐系统可以推荐个人生态系统。
- “个人说明书系统”不再作为推荐子系统出现。
- 用户界面没有 JSON 导出入口。
- 用户界面没有 Markdown 导出入口。
- 用户界面没有完整个人说明书编辑入口。
- 用户仍然可以重置本地数据。
- 本地优先原则保持不变。
- 不引入账号、云同步、云数据库或 AI 分析。
