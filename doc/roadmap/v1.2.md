# LifeOS v1.2 Roadmap

## 版本定位

LifeOS v1.2 是第一个真实子系统闭环版本。

v1.1 已经把 LifeOS 从“个人说明书原型”纠偏为“六个子系统驱动的本地人生操作系统启动器”。v1.2 不继续扩展 onboarding，也不恢复个人说明书、导出或云端能力，而是开启第一个可反复使用的子系统。

v1.2 的目标是：

> 建立个人生态系统的最小观察闭环。

个人生态系统在 v1.2 中不做健康管理、不做习惯打卡、不做诊断。它只帮助用户本地记录和回顾身体与生活环境是否支撑自己运行。

相关架构决策：

- `doc/decisions/0004-semantic-observation-events.md`

## 继承自 v1.1 的边界

v1.2 继续继承：

- 本地优先。
- 用户数据保存在用户自己的设备上。
- 不使用账号系统。
- 不使用云同步。
- 不使用云端数据库保存个人生活数据。
- 不使用 AI 分析、AI 推理、心理诊断、人格诊断或医疗建议。
- 技术栈继续使用 Next.js + React + TypeScript。
- 本地持久化继续使用 IndexedDB + Dexie。
- 界面语言使用简体中文。
- 保留复古 Macintosh 风格和旧电脑感。
- onboarding 仍然只是初始扫描和子系统推荐器。
- 个人说明书仍然是后期板块，不是当前主功能。

v1.2 不修改 `doc/v1.1/` 的版本边界。v1.1 已完成并保持为六系统启动器；v1.2 的新需求进入 roadmap 或后续正式版本目录。

## 核心闭环

v1.2 的核心使用路径：

1. 用户从 LifeOS 启动面板进入个人生态系统。
2. 用户看到当前生态状态。
3. 用户可以选择性更新某一个维度，而不是被要求填写整张表。
4. 每次更新生成一个本地“生态观察点”。
5. 用户查看今日生态轨迹。
6. 用户查看过去 1 天、7 天、15 天、30 天的生活晴雨表。

最小闭环是：

```text
进入个人生态系统
  -> 看到当前生态状态
  -> 选择性记录一个观察点
  -> 本地保存
  -> 查看今日生态轨迹
  -> 查看生活晴雨表
```

用户可以只查看，不记录。系统不能在进入页面时强制弹出记录表，也不能把记录做成连续打卡压力。

## 产品结构

### 面板 A: 当前生态状态

当前生态状态是个人生态系统的主面板。

它展示六个维度的最近状态：

1. 睡眠恢复
2. 作息节律
3. 身体状态
4. 饮食饮水
5. 活动舒展
6. 环境支撑

每个维度是一张小状态卡，包含：

- 维度名称。
- 当前或最近一次语义状态。
- 记录时间。
- 一个轻量的“更新”入口。

如果某个维度今天没有记录，应显示“今日未观察”。如果最近一次记录不是今天，可以显示最近记录日期，但必须避免暗示它代表今天。

### 面板 B: 今日生态轨迹

今日生态轨迹展示当天 0 点以后产生的观察点。

它不是强制折线图。v1.2 推荐使用时间轴或稀疏点轨迹：

- 按维度分组。
- 每个观察点显示语义状态和时刻。
- 没有记录的时刻保持空白。
- 不对空白时段做插值。
- 不用折线连接不连续的点，避免暗示系统知道中间状态。

“0 点以后清零”只表示今日轨迹面板切换到新的一天，不表示删除历史数据。历史观察点仍用于生活晴雨表。

### 面板 C: 生活晴雨表

生活晴雨表负责回顾过去一段时间的生态状态。

支持范围：

- 1 天
- 7 天
- 15 天
- 30 天

每个范围内，按维度展示：

- 观察次数。
- 平均内部状态。
- 低位观察次数。
- 最近一次观察。
- 一句确定性摘要。

生活晴雨表可以使用内部数值计算，但用户侧应以语义状态展示，例如：

```text
过去 7 天，睡眠恢复多落在「勉强够」到「偏少」之间。
身体状态记录 8 次，其中 3 次落在「疲惫」或更低。
环境支撑整体接近「可接受」，但有 2 次进入「拖累」或更低。
```

这些只能叫观察或线索，不能叫结论、诊断或建议。

## 生态维度

v1.2 第一版包含六个维度。维度顺序固定为：

1. 睡眠恢复
2. 作息节律
3. 身体状态
4. 饮食饮水
5. 活动舒展
6. 环境支撑

维度必须配置化。新增维度应优先通过配置完成，而不是复制一套业务逻辑。

### 语义选项与内部状态

用户选择语义状态。系统内部可以把语义状态映射为 `internalScore`，用于确定性统计。

内部状态建议使用 7 档：

```text
+3 很支撑
+2 支撑
+1 略微支撑
 0 中性或可用
-1 略微拖累
-2 拖累
-3 明显拖累
```

`没观察` 不是 7 档之一，也不应写入平均值。没有观察就是没有数据。

## 数据模型草案

v1.2 应新增个人生态系统的本地数据模型。

```ts
type EcosystemDimensionId =
  | "sleepRecovery"
  | "dailyRhythm"
  | "bodyState"
  | "foodWater"
  | "activityStretch"
  | "environmentSupport";

type EcosystemSemanticValue = {
  id: string;
  label: string;
  internalScore: -3 | -2 | -1 | 0 | 1 | 2 | 3;
};

type EcosystemDimensionDefinition = {
  id: EcosystemDimensionId;
  label: string;
  description: string;
  values: EcosystemSemanticValue[];
};

type EcosystemObservation = {
  id: string;
  dimensionId: EcosystemDimensionId;
  valueId: string;
  valueLabel: string;
  internalScore: -3 | -2 | -1 | 0 | 1 | 2 | 3;
  observedAt: string;
  note?: string;
};
```

实现时可以按现有类型风格调整命名，但必须保留这些概念：

- 维度 ID。
- 语义值 ID。
- 语义快照。
- 内部分数。
- 观察时间。
- 可选备注。

`valueLabel` 和 `internalScore` 应在观察点中保存快照，避免未来调整维度配置后，历史记录被重新解释。

## 本地时间规则

今日生态轨迹按浏览器本地日期分组。

原则：

- 今日从用户设备本地时间 00:00 开始。
- 1 天、7 天、15 天、30 天统计同样使用本地日期边界。
- 0 点后今日轨迹自然为空，但历史数据不删除。
- 不引入服务器时间。

## 聚合规则草案

v1.2 的聚合必须是本地确定性的，不使用 AI。

### 当前生态状态

每个维度取最近一次观察点。

展示时区分：

- 今日已观察。
- 最近观察不是今天。
- 从未观察。

### 今日生态轨迹

筛选 `observedAt` 位于本地今天范围内的观察点。

按维度分组，再按时间升序展示。

### 生活晴雨表

对指定范围内的观察点按维度分组。

每个维度计算：

- `observationCount`
- `averageInternalScore`
- `lowObservationCount`
- `latestObservation`
- `summaryLabel`

`lowObservationCount` 统计 `internalScore <= -1` 的观察点。v1.2 不要求单独展示 `internalScore <= -2` 的明显拖累次数。

`summaryLabel` 由平均值映射回该维度最接近的语义状态。它不是诊断，只是这段记录的平均位置。

## v1.2 不包含内容

v1.2 不包含：

- 健康建议。
- 医疗建议。
- 心理诊断。
- AI 分析。
- 云同步。
- 账号系统。
- 云端数据库。
- 用户直接编辑内部分数。
- 1 到 5 分或 1 到 10 分的显性评分 UI。
- 连续打卡、连续天数、惩罚或完成率压力。
- 复杂习惯系统。
- 家务清单系统。
- 其他五个子系统的真实业务闭环。
- 完整个人说明书。
- JSON 导出。
- Markdown 导出。
- Markdown 导入。
- 备份恢复。

## 与 v1.1 启动面板的关系

v1.1 启动面板中六个子系统入口已经存在。

v1.2 应将“个人生态系统”入口从“后续开放”推进为可进入状态。

其他五个子系统仍可保持后续开放。v1.2 不应为了导航完整性而提前实现其他子系统。

## 风险点

- 用户侧如果看到分数，产品气质会变成评价系统。UI 应展示语义，不展示内部分数。
- 折线图可能暗示系统知道连续状态。v1.2 应优先使用稀疏时间轴或点状轨迹。
- 每日强制记录会变成打卡压力。记录必须是选择性的。
- 0 点清零容易被误解为删除数据。实现和文案要明确只是今日视图换日。
- 维度过多会让用户费神。v1.2 固定六个必要维度即可。
- 过早扩展其他子系统会稀释 v1.2 的目标。v1.2 只开启个人生态系统。

## 最小可接受版本

如果时间紧，v1.2 的最小可接受版本是：

- 个人生态系统入口可进入。
- 六个生态维度可选择性记录。
- 每次记录保存为本地观察点。
- 当前生态状态能显示每个维度的最近观察。
- 今日生态轨迹能显示当天观察点。
- 生活晴雨表能显示 1 天和 7 天摘要。
- 不展示内部分数。
- 不强制记录。
- 不引入云端、账号、AI 分析或诊断。
