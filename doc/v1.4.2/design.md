# LifeOS v1.4.2 设计文档

## 1. 版本定位

LifeOS v1.4.2 是财务管理系统的第二个增量业务版本。

v1.4 已经开启财务管理系统，并完成“我的钱包”的本地余额快照闭环。v1.4.1 已经新增“收入来源”面板，完成“收入来源 -> 手动入账 -> 我的钱包更新”的闭环。v1.4.2 不修改 v1.4 或 v1.4.1 的历史边界，不把 v1.4.2 的新需求写回 `doc/v1.4/` 或 `doc/v1.4.1/`。

v1.4.2 在 v1.4.1 的基础上新增第三个真实财务面板：

> 日常开销池。

v1.4.2 不是完整记账系统、完整预算系统、完整财富流动日志、财务诊断系统或财务顾问。它只提供一个本地、手动、无审判的“钱包容器 -> 划入开销池 -> 消费扣减 -> 本地消费流水”的闭环。

v1.4.2 真正新增的可用业务只有：

- 日常开销池。

## 2. 继承自 v1.4.1 的原则

v1.4.2 继续继承这些原则：

- 本地优先。
- 用户数据保存在用户自己的设备上。
- 不使用账号系统。
- 不使用云同步。
- 不使用云端数据库保存个人生活数据。
- 不使用 AI 分析、AI 推理、财务诊断、投资诊断、职业判断、心理诊断或医疗建议。
- 技术栈继续使用 Next.js + React + TypeScript。
- 本地持久化继续使用 IndexedDB + Dexie。
- 界面语言使用简体中文。
- 保留复古 Macintosh 风格和旧电脑感。
- onboarding 仍然只是初始扫描和子系统推荐器。
- 个人说明书仍然是后期板块，不是当前主功能。
- 推荐、扫描线索、子系统展示和财务计算都必须是确定性逻辑，不得伪装成诊断、建议或自动判断。

v1.4.2 继续保持这些已完成能力：

- 个人生态系统 v1.2 闭环。
- 能量管理系统 v1.3 闭环。
- 启动面板和六个子系统入口。
- 财务管理系统可进入。
- “我的钱包”本地余额快照闭环。
- “收入来源”本地手动入账闭环。
- 重置本地数据时清空生态观察点、能量观察点、钱包容器和收入来源。

v1.4.2 不继承 v1.4.1 中“日常开销池业务不包含”的范围限制。v1.4.2 会把日常开销池从后续开放结构推进为当前可用面板。

## 3. v1.4.2 新增内容

### 3.1 日常开销池面板

v1.4.2 在财务管理系统页面新增“日常开销池”面板。

日常开销池用于让用户从“我的钱包”中的一个资金容器里划出一笔日常可用金额，然后在本地记录日常消费扣减。

用户可以自行决定一次划入的钱对应一天、一周、一个月、一年或其他周期。系统不过问、不保存周期字段、不根据周期判断用户行为。

日常开销池只有一个全局池，不支持多个开销池或分类开销池。

### 3.2 划入开销池

用户在日常开销池的主信息区选择当前划款来源钱包容器，并输入划入金额。

划入后：

- 当前选择的钱包容器余额减少。
- 日常开销池余额增加。
- 日常开销池保存最近一次划款金额。
- 日常开销池保存最近一次划款日期。
- 日常开销池保存最近一次划款来源容器名称快照。
- “我的钱包”的当前总余额和环形图按 v1.4 规则刷新。

v1.4.2 不保存完整划款历史。划款只更新开销池当前状态和最近一次划款信息。

### 3.3 消费结算

用户在消费结算区输入本次消费金额和备注，并点击立即扣款。

消费后：

- 日常开销池余额减少。
- 生成一条本地消费流水。
- 钱包容器不变化。

消费只扣开销池余额，不直接读取或修改钱包容器。钱包容器只在划入开销池时扣款。

### 3.4 消费流水

v1.4.2 保存日常开销池内部的本地消费流水。

每条消费流水包含：

- 金额。
- 备注。
- 消费时间。
- 创建时间。
- 更新时间。

消费流水用于展示用户在日常开销池中的扣减记录。它不是完整财富流动日志，不承担收入、储蓄、转账、梦想账户、金鹅账户或跨系统资金事件源职责。

### 3.5 删除消费流水

用户可以删除消费流水。

删除消费流水后：

- 对应金额加回日常开销池余额。
- 钱包容器不变化。
- 该条消费流水从本地存储中删除。

删除消费流水不修改钱包容器余额，因为钱包容器只在划入开销池时扣过一次。

## 6. 产品流程

v1.4.2 的首次使用路径继承 v1.4.1：

1. 用户在本地打开 LifeOS。
2. 如果没有本地扫描记录，展示启动界面和 onboarding。
3. 用户完成 9 道题。
4. LifeOS 保存扫描回答。
5. LifeOS 通过确定性规则生成初始扫描结果。
6. LifeOS 展示启动面板。
7. 启动面板展示六个子系统入口。
8. 用户点击财务管理系统入口。
9. LifeOS 展示财务管理系统页面。

v1.4.2 新增的日常开销池路径：

1. 用户进入财务管理系统。
2. 用户看到“我的钱包”“收入来源”和“日常开销池”。
3. 如果还没有钱包容器，日常开销池提示先创建钱包容器，并禁用划款入口。
4. 用户在“我的钱包”创建至少一个资金容器。
5. 用户在日常开销池上方主信息区选择当前划款来源容器。
6. 用户输入划入金额。
7. LifeOS 校验划入金额为正数，且不超过来源容器余额。
8. LifeOS 将来源容器余额减少，并将日常开销池余额增加。
9. LifeOS 保存最近一次划款金额、日期和来源容器名称快照。
10. 用户在消费结算区输入消费金额和必填备注。
11. LifeOS 校验消费金额为正数，且不超过开销池余额。
12. LifeOS 将开销池余额减少。
13. LifeOS 保存一条本地消费流水。
14. 用户可以删除消费流水。
15. 删除消费流水时，LifeOS 将对应金额加回开销池余额。

## 7. 界面设计

### 7.1 财务管理系统页面

页面主标题继续使用：

```text
财务管理系统
```

系统定义继续使用：

```text
资源、消费和自由度
```

核心区块：

1. 我的钱包。
2. 收入来源。
3. 日常开销池。
4. 财务系统未来结构。
5. 返回启动面板入口。

### 7.2 日常开销池面板

“日常开销池”展示用户手动维护的本地日常可用开销。

文案应明确这是本地手动开销池，例如：

```text
这是你手动划入、手动结算的本地日常开销池。
```

不得暗示系统连接了银行、支付平台或外部账户。

日常开销池面板内部采用上中下结构：

1. 上方小窗口：开销池主信息和划款入口。
2. 中间小窗口：消费结算。
3. 下方小窗口：消费流水。

### 7.3 上方主信息和划款入口

上方小窗口包含：

- 开销池当前余额。
- 当前划款来源钱包容器。
- 切换划款来源容器入口。
- 划入金额输入。
- 划款按钮。
- 最近一次划款金额。
- 最近一次划款日期。
- 最近一次划款来源容器名称快照。

没有钱包容器时：

- 提示先创建钱包容器。
- 划款入口禁用。
- 消费结算仍根据开销池余额判断是否可用。

当前划款来源容器缺失时：

- 开销池余额保留。
- 消费流水保留。
- 消费结算仍然可用，只要开销池余额足够。
- 划款入口禁用。
- UI 提示用户重新选择划款来源容器。
- 最近一次划款来源容器名称快照仍可展示为历史信息。

### 7.4 中间消费结算

中间小窗口包含：

- 消费金额输入。
- 消费备注输入。
- 立即扣款按钮。

要求：

- 消费金额必填。
- 消费金额必须为正数。
- 消费金额不能超过开销池余额。
- 备注必填。
- 备注首尾空格会被裁剪，裁剪后不能为空。
- 余额不足时不能扣款。
- UI 可以显示余额不足的克制提示。
- UI 不使用预算警报、超支惩罚或财务诊断文案。

### 7.5 下方消费流水

下方小窗口包含：

- 消费流水列表。
- 每条流水的金额。
- 每条流水的备注。
- 每条流水的消费时间。
- 每条流水的删除入口。

删除入口可以放在每条流水项中。删除前是否要求确认由实现阶段按现有 UI 风格决定，但删除后必须回滚开销池余额。

## 8. 数据模型草案

### 8.1 日常开销池

```ts
type DailyExpensePool = {
  id: "default";
  balance: number;
  selectedWalletContainerId?: string;
  lastTransferAmount?: number;
  lastTransferAt?: string;
  lastTransferWalletContainerId?: string;
  lastTransferWalletContainerNameSnapshot?: string;
  createdAt: string;
  updatedAt: string;
};
```

说明：

- `id` 固定为 `"default"`，表示 v1.4.2 只有一个全局日常开销池。
- `balance` 是开销池当前余额。
- `selectedWalletContainerId` 是当前划款来源容器。
- `lastTransferAmount` 是最近一次划款金额。
- `lastTransferAt` 是最近一次划款时间。
- `lastTransferWalletContainerId` 是最近一次划款来源容器 ID。
- `lastTransferWalletContainerNameSnapshot` 是最近一次划款来源容器名称快照。

### 8.2 消费流水

```ts
type DailyExpenseEntry = {
  id: string;
  amount: number;
  note: string;
  spentAt: string;
  createdAt: string;
  updatedAt: string;
};
```

说明：

- `amount` 必须为正数，展示时可以用消费方向文案或负号视觉表达。
- `note` 必填。
- `spentAt` 是消费发生时间，v1.4.2 默认使用用户确认扣款时的本地时间。

### 8.3 划款来源状态

划款来源状态不需要写入数据库，可以由当前钱包容器列表即时计算：

```ts
type DailyExpenseTransferSourceStatus = "available" | "missing" | "unselected";
```

如果 `selectedWalletContainerId` 在当前钱包容器列表中存在，则为 `available`。如果有值但当前钱包容器列表中不存在，则为 `missing`。如果没有选择来源容器，则为 `unselected`。

## 9. 业务规则

### 9.1 划款规则

```text
用户选择当前划款来源容器
  -> 输入划入金额
  -> amount > 0
  -> source.balance >= amount
  -> source.balance = source.balance - amount
  -> pool.balance = pool.balance + amount
  -> pool.lastTransferAmount = amount
  -> pool.lastTransferAt = now
  -> pool.lastTransferWalletContainerId = source.id
  -> pool.lastTransferWalletContainerNameSnapshot = source.name
  -> 保存 source
  -> 保存 pool
  -> 刷新我的钱包汇总
```

划款不生成划款历史，不生成完整财富流动日志事件。

### 9.2 消费规则

```text
用户输入消费金额和备注
  -> amount > 0
  -> note.trim().length > 0
  -> pool.balance >= amount
  -> pool.balance = pool.balance - amount
  -> 创建 DailyExpenseEntry
  -> 保存 pool
  -> 保存 entry
```

消费不修改钱包容器余额。

### 9.3 删除流水规则

```text
用户删除某条消费流水
  -> 找到 entry
  -> pool.balance = pool.balance + entry.amount
  -> 删除 entry
  -> 保存 pool
```

删除流水不修改钱包容器余额。

### 9.4 余额不足规则

来源容器余额不足时：

- 不能划入。
- 显示克制提示。
- 不出现“警报”“危险”“你应该”等文案。

开销池余额不足时：

- 不能消费。
- 显示克制提示。
- 不出现“超支惩罚”“预算失败”“财务风险”等文案。

## 12. 已确认决策

以下决策适用于 v1.4.2：

1. v1.4.2 继续使用 Next.js + React + TypeScript。
2. 本地持久化继续使用 IndexedDB + Dexie。
3. 本地优先原则不变。
4. 六个子系统是 LifeOS 的长期主体。
5. v1.4.2 保持个人生态系统已完成闭环。
6. v1.4.2 保持能量管理系统已完成闭环。
7. v1.4.2 保持财务管理系统可进入。
8. v1.4.2 保持“我的钱包”余额快照闭环。
9. v1.4.2 保持“收入来源”手动入账闭环。
10. v1.4.2 新增“日常开销池”真实业务面板。
11. v1.4.2 只做一个全局日常开销池。
12. v1.4.2 的划款来源容器在主信息区选择，切换后只影响下一次划款。
13. v1.4.2 划款时钱包容器余额减少，开销池余额增加。
14. v1.4.2 不保存完整划款历史，只保存最近一次划款信息。
15. v1.4.2 消费只扣开销池余额，不修改钱包容器余额。
16. v1.4.2 消费必须填写金额和备注。
17. v1.4.2 保存本地消费流水。
18. v1.4.2 删除消费流水时回滚开销池余额，不修改钱包容器余额。
19. v1.4.2 不做周期判断、提醒、预测、预算警报、超支惩罚、财务诊断、账号、云同步或云端数据库。
