# LifeOS v1.4.2 Agent Prompt

## 目标

根据当前项目文档，实现 LifeOS v1.4.2。

v1.4.2 是财务管理系统的第二个增量业务版本。它建立在 v1.4 已完成的“我的钱包”本地余额快照闭环和 v1.4.1 已完成的“收入来源”手动入账闭环之上，新增第三个真实财务面板：

> 日常开销池。

v1.4.2 的新增闭环是：

```text
进入财务管理系统
  -> 看到“日常开销池”
  -> 在主信息区选择当前划款来源钱包容器
  -> 输入划入金额
  -> 钱包容器余额减少
  -> 日常开销池余额增加
  -> 用户输入消费金额和必填备注
  -> 开销池余额减少
  -> 生成本地消费流水
  -> 删除消费流水时回滚开销池余额
```

实现过程应支持多 agent 协作。主 Agent 负责整体集成、任务分派、进度跟踪和最终验证；子 Agent 按 `doc/v1.4.2/tasks/` 中的任务文件领取模块。

## 数据模型要求

日常开销池建议模型：

```ts
type DailyExpensePool = {
  id: "default";
  balance: number;
  selectedWalletContainerId?: string;
  lastTransferAmount?: number;
  lastTransferAt?: string;
  lastTransferWalletContainerId?: string;
  lastTransferWalletContainerNameSnapshot?: string;
  createdAt: string;
  updatedAt: string;
};
```

消费流水建议模型：

```ts
type DailyExpenseEntry = {
  id: string;
  amount: number;
  note: string;
  spentAt: string;
  createdAt: string;
  updatedAt: string;
};
```

实现时可以按现有类型风格调整命名，但必须保留这些概念：

- 单一开销池。
- 当前余额。
- 当前划款来源钱包容器 ID。
- 最近一次划款金额。
- 最近一次划款时间。
- 最近一次划款来源容器 ID。
- 最近一次划款来源容器名称快照。
- 消费流水 ID。
- 消费金额。
- 消费备注。
- 消费时间。
- 创建时间。
- 更新时间。

不要新增：

- `DailyExpensePoolCategory`。
- `DailyExpensePoolPeriod`。
- `DailyExpenseBudget`。
- `DailyExpenseTransferHistory`。
- `Transaction`。
- `LedgerEntry`。
- `WealthFlowEvent`。

划款来源状态不需要写入数据库。它应由当前钱包容器列表即时计算：

```ts
type DailyExpenseTransferSourceStatus = "available" | "missing" | "unselected";
```

## 划款规则

```text
用户选择当前划款来源容器
  -> 输入划入金额
  -> 校验 amount > 0
  -> 校验来源容器存在
  -> 校验 source.balance >= amount
  -> source.balance = source.balance - amount
  -> source.updatedAt = now
  -> pool.balance = pool.balance + amount
  -> pool.lastTransferAmount = amount
  -> pool.lastTransferAt = now
  -> pool.lastTransferWalletContainerId = source.id
  -> pool.lastTransferWalletContainerNameSnapshot = source.name
  -> pool.updatedAt = now
  -> 保存 source
  -> 保存 pool
  -> 重新读取或刷新钱包容器
  -> 使用现有钱包汇总规则刷新总余额和环形图
```

来源容器缺失：

- 开销池继续显示。
- 消费流水继续显示。
- 消费结算仍然可用，只要开销池余额足够。
- 划款入口禁用。
- 显示“需要重新选择划款来源容器”或同等含义的克制文案。
- 不删除开销池。
- 不删除消费流水。
- 不自动选择其他钱包容器。

## 消费规则

```text
用户输入消费金额和备注
  -> 校验 amount > 0
  -> 校验 note.trim().length > 0
  -> 校验 pool.balance >= amount
  -> pool.balance = pool.balance - amount
  -> pool.updatedAt = now
  -> 创建 DailyExpenseEntry
  -> 保存 pool
  -> 保存 entry
```

开销池余额不足：

- 消费按钮不能成功扣款。
- 可以显示克制提示。
- 不生成消费流水。
- 不修改开销池余额。
- 不修改钱包容器余额。

## 删除消费流水规则

```text
用户删除某条消费流水
  -> 找到 entry
  -> pool.balance = pool.balance + entry.amount
  -> pool.updatedAt = now
  -> 删除 entry
  -> 保存 pool
```

删除消费流水：

- 回滚开销池余额。
- 不修改钱包容器余额。
- 不创建反向流水。
- 不创建退款流水。

## 产品约束

LifeOS v1.4.2 必须保持本地优先：

- 用户数据只保存在用户自己的设备上。
- 默认不向任何云端上传个人生活数据。
- 不实现账号系统、云同步或多人协作。
- 不使用 AI 分析、AI 推理、财务诊断、投资诊断、职业判断、心理诊断或医疗诊断。
- 初始扫描、生态观察、能量观察、钱包汇总、收入来源入账和日常开销池计算都由确定性逻辑完成。

日常开销池必须保持手动和非审判式：

- 用户手动选择划款来源。
- 用户手动输入划入金额。
- 用户手动输入消费金额。
- 用户手动填写消费备注。
- 用户自己决定划入金额对应什么周期。
- 系统不保存周期字段。
- 系统不判断用户的日常开销是否合理。
- 系统不生成预算建议、消费建议、提醒、预测、警报或诊断。
- UI 不告诉用户应该怎么花钱、存钱或投资。
- UI 不暗示连接了银行、支付平台或外部账户。
- UI 不用“警报”“危险”“你应该”“你必须”等训诫文案。

界面语言使用简体中文。

## 并行开发建议

推荐顺序：

1. 先完成 `daily-expense-local-data.md`。
2. `daily-expense-actions.md` 可以在类型和存储函数稳定后开始。
3. `daily-expense-panel.md` 可以在类型稳定后开始，动作函数稳定后接入真实行为。
4. `finance-management-integration.md` 在面板和动作行为可用后收口。
5. `final-verification.md` 最后执行。

并行注意事项：

- 不要让多个 Agent 同时改同一个组件的大段结构。
- 如果需要拆分组件，优先让面板 Agent 新增独立组件，再由集成 Agent 接入。
- 类型定义只能有一个事实源。
- 本地存储函数只能有一个事实源。
- 划款、消费和删除流水动作只能有一个事实源。
- 任何 Agent 都不能为了完成自己的任务提前实现周期判断、预算系统、完整财富流动日志、退款流水、划款历史、梦想账户或金鹅账户。

## 进度更新规则

完成模块后，更新 `doc/v1.4.2/tasks/progress.md`。

一个模块只有在以下条件都满足时，才能标记完成：

- 对应任务文件中的 checklist 全部完成。
- 模块通过自己的单元测试或组件测试。
- 模块没有引入云端依赖。
- 模块没有重新引入 v1.4.2 明确排除的用户侧功能。
- 模块边界符合 `doc/v1.4.2/design.md`。

不要提前把未完成模块标记为完成。

## 完成标准

v1.4.2 完成时必须满足：

- `doc/v1.4.2/tasks/progress.md` 所有任务完成。
- 所有测试通过。
- TypeScript 类型检查通过。
- lint 通过。
- build 通过。
- 本地页面可完成首次扫描。
- 完成扫描后进入 LifeOS 启动面板。
- 点击个人生态系统可以进入个人生态系统页面。
- 点击能量管理系统可以进入能量管理系统页面。
- 点击财务管理系统可以进入财务管理系统页面。
- 财务管理系统展示“我的钱包”“收入来源”和“日常开销池”。
- 用户可以从钱包容器划入日常开销池。
- 划款后钱包容器余额减少。
- 划款后开销池余额增加。
- 划款后钱包总余额和环形图刷新。
- 用户可以输入消费金额和备注进行扣款。
- 消费后开销池余额减少。
- 消费后生成本地消费流水。
- 消费不修改钱包容器余额。
- 用户可以删除消费流水。
- 删除消费流水后开销池余额加回。
- 删除消费流水不修改钱包容器余额。
- 当前划款来源容器缺失时，开销池保留，消费仍可用，划款入口禁用。
- 重置后可以回到首次启动状态，并清空生态观察点、能量观察点、钱包容器、收入来源、日常开销池和消费流水。
- UI 没有预算警报、超支惩罚、财务健康评分、消费建议、投资建议、职业建议或财务诊断文案。
- 用户界面没有 JSON 导出。
- 用户界面没有 Markdown 导出。
- 用户界面没有完整个人说明书编辑入口。
- 不保存划款历史。
- 不做周期判断、不提醒、不预测。
- 不引入账号、云同步、云数据库、银行连接、支付连接或 AI 分析。
