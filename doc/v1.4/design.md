# LifeOS v1.4 设计文档

## 1. 版本定位

LifeOS v1.4 是第三个真实子系统版本。

v1.2 已经开启个人生态系统并完成最小观察闭环。v1.3 已经开启能量管理系统并完成最小观察闭环。v1.4 不继续扩展 onboarding，不恢复个人说明书、导出或云端能力，也不修改 v1.3 的历史边界，而是开启第三个可进入的子系统：财务管理系统。

v1.4 的目标是：

> 建立财务管理系统的基础设施版本。

这个版本不是完整理财系统、记账系统、预算系统、投资系统或财务顾问。它只提供一个本地、手动、无审判的“我的钱包”闭环，让用户看见自己的资金容器、当前总余额和资源分布。

v1.4 真正可用的业务只有：

- 我的钱包。

财务管理系统页面可以展示未来结构占位，但金鹅账户、梦想账户、金钱流入来源、日常开销池和财富流动日志不在 v1.4 实现业务闭环。

## 2. 继承自 v1.3 的原则

v1.4 继续继承这些原则：

- 本地优先。
- 用户数据保存在用户自己的设备上。
- 不使用账号系统。
- 不使用云同步。
- 不使用云端数据库保存个人生活数据。
- 不使用 AI 分析、AI 推理、财务诊断、投资诊断、心理诊断或医疗建议。
- 技术栈继续使用 Next.js + React + TypeScript。
- 本地持久化继续使用 IndexedDB + Dexie。
- 界面语言使用简体中文。
- 保留复古 Macintosh 风格和旧电脑感。
- onboarding 仍然只是初始扫描和子系统推荐器。
- 个人说明书仍然是后期板块，不是当前主功能。
- 推荐、扫描线索和子系统展示都必须是确定性逻辑，不得伪装成诊断、建议或自动判断。

v1.4 不继承 v1.3 中“财务管理系统仍保持后续开放”的范围限制。v1.4 会把财务管理系统推进为可进入状态。认知管理系统、人生目标管理系统和人际关系管理系统仍保持后续开放。

## 3. v1.4 新增内容

### 3.1 财务管理系统基础设施

v1.4 新增财务管理系统页面。

最小闭环是：

```text
进入财务管理系统
  -> 看到“我的钱包”
  -> 新建资金容器
  -> 填写当前余额
  -> 本地保存
  -> 当前总余额和环形图更新
  -> 用户可以直接编辑余额
```

用户可以只查看，不新增容器。系统不能强制用户填写完整资产结构，也不能把财务状态包装成评分、警报或建议。

### 3.2 我的钱包

“我的钱包”是 v1.4 唯一真正可用的业务。

它是一个手动余额快照工具：

- 用户创建若干资金容器。
- 用户直接填写或编辑每个容器的当前余额。
- 系统汇总当前总余额。
- 总余额为正时展示彩色环形分布。
- 总余额为零或负时展示灰色状态，不展示占比。

v1.4 使用余额快照模型，不使用流水账模型。系统不追踪每一笔变化。

### 3.3 资金容器

资金容器是财务管理系统的核心本地数据对象。

每个资金容器包含：

- 名称。
- 当前余额。
- 颜色。
- 可选备注。
- 创建时间。
- 更新时间。

v1.4 不给资金容器做分类。资金容器不区分银行卡、支付钱包、现金、信用负债、梦想账户或金鹅账户。

### 3.4 钱包图表状态

v1.4 新增钱包图表状态规则：

- 总余额为正：展示彩色环形分布。
- 总余额为零：展示灰色状态，不展示占比。
- 总余额为负：展示灰色状态，不展示占比，并显示当前负债金额。

负数容器保留在容器列表中显示真实余额，但不参与正余额占比计算。

## 4. v1.4 修改内容

### 4.1 启动面板入口

v1.3 启动面板中个人生态系统和能量管理系统已经可进入，财务管理系统仍是后续开放入口。v1.4 将财务管理系统入口从“后续开放”推进为可进入状态。

要求：

- 点击财务管理系统进入财务管理系统页面。

- 能量管理系统观察点。

重置本地数据时，应同时清空个人生态系统观察点、能量管理系统观察点和钱包容器。

## 6. 产品流程

v1.4 的首次使用路径继承 v1.3：

1. 用户在本地打开 LifeOS。
2. 如果没有本地扫描记录，展示启动界面和 onboarding。
3. 用户完成 9 道题。
4. LifeOS 保存扫描回答。
5. LifeOS 通过确定性规则生成初始扫描结果。
6. LifeOS 展示启动面板。
7. 启动面板展示六个子系统入口。

v1.4 新增的财务管理系统路径：

1. 用户点击财务管理系统入口。
2. LifeOS 展示财务管理系统页面。
3. 用户看到“我的钱包”。
4. 用户可以新建资金容器。
5. 用户填写容器名称、当前余额、颜色和可选备注。
6. LifeOS 保存钱包容器到 IndexedDB。
7. 页面更新当前总余额。
8. 页面根据总余额正负展示彩色分布或灰色状态。
9. 用户可以直接编辑容器当前余额。
10. 用户可以删除资金容器。
11. 用户可以返回 LifeOS 启动面板。

## 7. 界面设计

### 7.1 财务管理系统页面

页面主标题：

```text
财务管理系统
```

系统定义：

```text
资源、消费和自由度
```

核心区块：

1. 我的钱包。
2. 财务系统未来结构。
3. 返回启动面板入口。

### 7.2 我的钱包

“我的钱包”展示用户手动维护的钱包快照。

它包含：

- 当前总余额或当前负债金额。
- 钱包环形图。
- 资金容器图例。
- 资金容器列表。
- 新建资金容器入口。

文案应明确这是本地手动余额快照，例如：

```text
这是你手动记录的本地余额快照。
```

不得暗示系统连接了银行、支付平台或外部账户。

### 7.3 资金容器编辑

用户可以新建、编辑和删除资金容器。

新建或编辑时包含：

- 名称。
- 当前余额。
- 颜色。
- 可选备注。

要求：

- 当前余额允许为正数、零或负数。
- 保存后更新 `updatedAt`。
- 删除容器后重新计算总余额。
- UI 不要求用户填写容器分类。
- UI 不要求用户记录流水。
- UI 不使用预算警报、超支惩罚或财务诊断文案。

### 7.4 正余额状态

当 `totalBalance > 0`：

- 环形图正常展示。
- 只使用正余额容器计算占比。
- 中间显示：

```text
当前总余额
```

- 图例展示参与分布的容器名称、颜色、金额和占比。
- 余额为 0 或负数的容器不参与占比计算，但仍在容器列表中显示真实余额。

### 7.5 零余额状态

当 `totalBalance === 0`：

- 不展示占比。
- 环形图使用中性灰色空环或占位环。
- 中间显示：

```text
当前总余额
0
```

- 可以显示：

```text
还没有可分配余额快照
```

### 7.6 负余额状态

当 `totalBalance < 0`：

- 环形图整体变灰。
- 不展示各容器占比。
- 中间显示：

```text
当前负债金额
```

- 容器列表仍然显示每个容器的真实余额。
- 可以显示：

```text
当前总和为负，暂不计算资金占比。
```

- 不出现“警报”“危险”“你超支了”“你应该”等文案。
- 当用户更新容器后 `totalBalance > 0`，自动恢复正常彩色环形图和占比。

### 7.7 财务系统未来结构

财务管理系统页面可以展示这些后续开放区块：

- 金鹅账户。
- 梦想账户。
- 金钱流入来源。
- 日常开销池。
- 财富流动日志。

要求：

- 只能展示后续开放状态。
- 不能提供编辑入口。
- 不能保存业务数据。
- 不能提前实现业务计算。
- 不能把未来结构写成当前已支持能力。

## 8. 数据模型草案

### 8.1 钱包容器

```ts
type WalletContainer = {
  id: string;
  name: string;
  balance: number;
  color: string;
  note?: string;
  createdAt: string;
  updatedAt: string;
};
```

### 8.2 钱包汇总

```ts
type WalletSummaryStatus = "positive" | "zero" | "negative";

type WalletDistributionItem = {
  containerId: string;
  name: string;
  balance: number;
  color: string;
  percentage: number;
};

type WalletSummary = {
  totalBalance: number;
  status: WalletSummaryStatus;
  distributionItems: WalletDistributionItem[];
};
```

`distributionItems` 只包含正余额容器，并且只在 `totalBalance > 0` 时用于展示占比。

## 9. 聚合规则

### 9.1 当前总余额

当前总余额由所有资金容器加总得到：

```text
totalBalance = sum(container.balance)
```

### 9.2 状态判断

```text
totalBalance > 0  -> positive
totalBalance == 0 -> zero
totalBalance < 0  -> negative
```

### 9.3 占比计算

仅当 `totalBalance > 0` 时计算占比。

占比只使用正余额容器：

```text
positiveContainers = containers.filter(balance > 0)
percentage = container.balance / sum(positiveContainers.balance)
```

如果存在负数容器，负数容器不参与占比计算，但仍保留在列表中。

如果 `totalBalance <= 0`，不计算占比，不展示分布百分比。

## 12. 已确认决策

以下决策适用于 v1.4：

1. v1.4 继续使用 Next.js + React + TypeScript。
2. 本地持久化继续使用 IndexedDB + Dexie。
3. 本地优先原则不变。
4. 六个子系统是 LifeOS 的长期主体。
5. v1.4 保持个人生态系统已完成闭环。
6. v1.4 保持能量管理系统已完成闭环。
7. v1.4 开启财务管理系统的基础设施版本。
8. v1.4 真正可用的财务业务只有“我的钱包”。
9. v1.4 使用余额快照模型，不使用流水账模型。
10. v1.4 不做资金容器分类。
11. v1.4 不做梦想账户、金鹅账户、金钱流入来源、日常开销池或财富流动日志业务。
12. v1.4 不使用 AI 分析、诊断、建议、账号、云同步或云端数据库。
