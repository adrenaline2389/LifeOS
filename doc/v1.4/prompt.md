# LifeOS v1.4 Agent Prompt

## 目标

根据当前项目文档，实现 LifeOS v1.4。

v1.4 是第三个真实子系统版本。目标是在 v1.3 的个人生态系统和能量管理系统闭环基础上，开启“财务管理系统”的基础设施版本。

实现过程应支持多 agent 协作。主 Agent 负责整体集成、任务分派、进度跟踪和最终验证；子 Agent 按 `doc/v1.4/tasks/` 中的任务文件领取模块。

## 输入

必须先阅读：

1. `doc/current.md`
2. `doc/README.md`
3. `doc/decisions/`
4. `doc/v1.4/design.md`
5. `doc/v1.4/tasks/progress.md`
6. `doc/v1.4/tasks/`

必要时参考：

- `doc/v1.3/design.md`
- `doc/v1.3/tasks/`
- `doc/roadmap/v1.4.md`

参考 v1.3 时，只继承仍然有效的本地优先、技术栈、启动面板、六系统主体、个人生态系统已完成闭环、能量管理系统已完成闭环、模块边界和测试原则。不要把 v1.3 中“财务管理系统仍保持后续开放”的范围限制带入 v1.4。

## 输出

v1.4 实现后应具备：

- v1.3 的首次启动扫描流程。
- v1.3 的 LifeOS 启动面板。
- v1.3 的个人生态系统闭环。
- v1.3 的能量管理系统闭环。

## 技术要求

- 应用框架：Next.js。
- 前端框架：React。
- 开发语言：TypeScript。
- 本地数据：IndexedDB。
- IndexedDB 封装：Dexie。
- 测试：沿用当前项目测试方案。
- 包管理器：沿用当前项目配置。

必须通过：

```bash
npm run test
npm run typecheck
npm run lint
```

最终发布前还必须通过：

```bash
npm run build
```

不要引入：

- 后端服务。
- 账号系统。
- 云同步。
- 云端数据库。
- 银行或支付平台连接。
- 外部 AI 服务。
- 用于分析、推理、建议或诊断用户的 AI 能力。

## 核心产品约束

LifeOS v1.4 必须保持本地优先：

- 用户数据只保存在用户自己的设备上。
- 默认不向任何云端上传个人生活数据。
- 不实现账号系统、云同步或多人协作。
- 不使用 AI 分析、AI 推理、财务诊断、投资诊断、心理诊断或医疗诊断。
- 初始扫描、生态观察、能量观察和钱包汇总都由确定性规则生成。

财务管理系统必须保持非审判式：

- 用户手动维护资金容器。
- 系统只展示余额快照和分布。
- UI 不使用预算警报、超支惩罚或财务健康评分。
- UI 不告诉用户应该怎么花钱、存钱或投资。
- UI 不暗示连接了银行、支付平台或外部账户。

界面语言使用简体中文。

## 模块边界

严格按照 `doc/v1.4/design.md` 和 `doc/v1.4/tasks/` 中的模块职责实现。

### App Shell

负责高层流程切换：

- 无本地扫描结果时展示 onboarding。
- 有本地扫描结果时展示 LifeOS 启动面板。
- 可以从启动面板进入个人生态系统。
- 可以从启动面板进入能量管理系统。
- 可以从启动面板进入财务管理系统。
- 可以从财务管理系统返回启动面板。
- 重置后回到首次启动状态。

不包含钱包容器存储、钱包汇总规则或钱包 UI 内部状态。

### Startup Dashboard

负责展示 LifeOS 启动面板：

- 继续展示六个子系统。
- 个人生态系统入口仍可进入。
- 能量管理系统入口仍可进入。
- 财务管理系统入口可进入。
- 财务管理系统被推荐时，推荐卡片可进入财务管理系统。
- 认知管理系统、人生目标管理系统和人际关系管理系统继续后续开放。
- 不展示 JSON / Markdown 导出入口。
- 不展示完整个人说明书编辑入口。

### Finance Management

负责财务管理系统页面：

- 展示页面标题 `财务管理系统`。
- 展示系统定义 `资源、消费和自由度`。
- 展示“我的钱包”。
- 允许用户新建、编辑和删除资金容器。
- 展示当前总余额。
- 展示钱包环形图。
- 展示财务系统未来结构占位。
- 提供返回启动面板入口。

### Wallet Insights

负责确定性聚合：

- 当前总余额。
- 钱包状态。
- 正余额分布占比。

不包含 React UI、IndexedDB 读写、AI 分析、建议或诊断文案。

### Local Data

负责本地保存、读取和重置：

- 保存 onboarding 回答。
- 保存启动扫描结果。
- 保存个人生态系统观察点。
- 保存能量管理系统观察点。
- 保存财务管理系统钱包容器。
- 删除钱包容器。
- 清空本地 LifeOS 数据。

不作为用户侧能力提供 JSON 或 Markdown 导出。

## Agent 工作方式

主 Agent 负责：

- 阅读 v1.4 全部输入文档。
- 检查当前代码与 v1.4 目标的差异。
- 按 `doc/v1.4/tasks/` 分派子 Agent。
- 确保并行任务不会互相覆盖。
- 审核子 Agent 输出。
- 集成模块。
- 更新 `doc/v1.4/tasks/progress.md`。
- 运行最终验证。

子 Agent 负责：

- 只处理领取的任务文件。
- 补充或更新对应测试。
- 保持模块边界清晰。
- 不修改无关模块。
- 不引入云端依赖。
- 不重复定义核心类型。
- 不把个人说明书、导入导出、AI 能力、建议能力、诊断能力、梦想账户、金鹅账户或流水账重新引入 v1.4。

## 并行开发建议

推荐顺序：

1. 先完成钱包容器类型与本地存储。
2. 同步或随后完成钱包汇总与图表状态规则。
3. 再实现财务管理系统页面 UI。
4. 再接入 App Shell 与启动面板入口。
5. 最后做文档同步和全量验收。

## 完成标准

v1.4 完成时必须满足：

- `doc/v1.4/tasks/progress.md` 所有任务完成。
- 所有测试通过。
- TypeScript 类型检查通过。
- lint 通过。
- build 通过。
- 本地页面可完成首次扫描。
- 完成扫描后进入 LifeOS 启动面板。
- 点击个人生态系统可以进入个人生态系统页面。
- 点击能量管理系统可以进入能量管理系统页面。
- 点击财务管理系统可以进入财务管理系统页面。
- 财务管理系统展示“我的钱包”。
- 用户可以新建资金容器。
- 用户可以直接编辑资金容器当前余额。
- 用户可以删除资金容器。
- 总余额为正时展示彩色环形分布。
- 总余额为零时展示灰色状态，不展示占比。
- 总余额为负时展示灰色状态，不展示占比，并显示当前负债金额。
- UI 没有预算警报、超支惩罚、财务健康评分、投资建议或财务诊断文案。
- 认知管理系统、人生目标管理系统和人际关系管理系统仍显示后续开放。
- 用户界面没有 JSON 导出。
- 用户界面没有 Markdown 导出。
- 用户界面没有完整个人说明书编辑入口。
- 重置后可以回到首次启动状态，并清空生态观察点、能量观察点和钱包容器。
