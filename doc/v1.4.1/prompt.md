# LifeOS v1.4.1 Agent Prompt

## 目标

根据当前项目文档，实现 LifeOS v1.4.1。

v1.4.1 是财务管理系统的第一个增量业务版本。它建立在 v1.4 已完成的“我的钱包”本地余额快照闭环之上，新增第二个真实财务面板：

> 收入来源。

v1.4.1 的新增闭环是：

```text
进入财务管理系统
  -> 看到“收入来源”
  -> 新增收入来源
  -> 绑定一个钱包容器
  -> 用户手动确认一次入账
  -> 对应钱包容器余额增加
  -> 我的钱包总额和环形图更新
```

实现过程应支持多 agent 协作。主 Agent 负责整体集成、任务分派、进度跟踪和最终验证；子 Agent 按 `doc/v1.4.1/tasks/` 中的任务文件领取模块。

## v1.4.1 必须实现

完成 LifeOS v1.4.1 的可运行实现。

v1.4.1 实现后应具备：

- v1.4 的首次启动扫描流程。
- v1.4 的 LifeOS 启动面板。
- v1.4 的个人生态系统闭环。
- v1.4 的能量管理系统闭环。
- v1.4 的财务管理系统页面。
- v1.4 的“我的钱包”业务。
- “我的钱包”仍支持新建、编辑和删除资金容器。
- “我的钱包”仍支持直接编辑资金容器当前余额。
- “我的钱包”仍按 v1.4 规则展示正余额、零余额和负余额状态。
- 财务管理系统新增“收入来源”面板。
- 用户可以新建、编辑和删除收入来源。
- 收入来源名称必填。
- 收入来源金额模式支持固定金额和不固定。
- 固定金额必须是正数。
- 收入来源频率模式支持日结、周结、月结、季结、年结和不固定。
- 收入来源必须绑定一个已有钱包容器。
- 没有钱包容器时，不能新增收入来源。
- 固定金额来源可以手动入账。
- 不固定金额来源手动入账时，用户必须输入本次到账金额。
- 手动入账后更新目标钱包容器余额和 `updatedAt`。
- 入账后“我的钱包”当前总余额和环形图刷新。
- 绑定的钱包容器被删除后，收入来源保留，但不能入账，并提示重新选择流入钱包。
- 删除收入来源不影响钱包容器余额。
- 重置本地数据时清空收入来源。

## 数据模型要求

收入来源建议模型：

```ts
type MoneyAmountPattern =
  | {
      kind: "fixed";
      amount: number;
    }
  | {
      kind: "variable";
    };

type MoneyFrequencyPattern =
  | {
      kind: "fixed";
      interval: "daily" | "weekly" | "monthly" | "quarterly" | "yearly";
    }
  | {
      kind: "variable";
    };

type MoneyInflowSource = {
  id: string;
  name: string;
  amountPattern: MoneyAmountPattern;
  frequencyPattern: MoneyFrequencyPattern;
  targetWalletContainerId: string;
  note?: string;
  createdAt: string;
  updatedAt: string;
};
```

实现时可以按现有类型风格调整命名，但必须保留这些概念：

- 来源 ID。
- 来源名称。
- 金额模式。
- 频率模式。
- 目标钱包容器 ID。
- 可选备注。
- 创建时间。
- 更新时间。

不要新增：

- `stability` 字段。
- `nextExpectedDate` 字段。
- `MoneyInflowEvent` 类型。
- `Transaction` 类型。
- `LedgerEntry` 类型。

目标钱包状态不需要写入数据库。它应由当前钱包容器列表即时计算：

```ts
type MoneyInflowTargetStatus = "available" | "missing";
```

## 手动入账规则

固定金额来源：

```text
amountPattern.kind === "fixed"
  -> amount = amountPattern.amount
  -> 校验目标钱包容器存在
  -> target.balance = target.balance + amount
  -> target.updatedAt = now
  -> 保存 target
  -> 重新读取或刷新钱包容器
  -> 使用现有钱包汇总规则刷新总余额和环形图
```

不固定金额来源：

```text
amountPattern.kind === "variable"
  -> 弹出或展示本次到账金额输入
  -> 用户输入 amount
  -> 校验 amount > 0
  -> 校验目标钱包容器存在
  -> target.balance = target.balance + amount
  -> target.updatedAt = now
  -> 保存 target
  -> 重新读取或刷新钱包容器
  -> 使用现有钱包汇总规则刷新总余额和环形图
```

目标容器缺失：

- 收入来源继续显示。
- 手动入账按钮禁用。
- 显示“需要重新选择流入钱包”或同等含义的克制文案。
- 不删除收入来源。
- 不自动选择其他钱包容器。

## 产品约束

LifeOS v1.4.1 必须保持本地优先：

- 用户数据只保存在用户自己的设备上。
- 默认不向任何云端上传个人生活数据。
- 不实现账号系统、云同步或多人协作。
- 不使用 AI 分析、AI 推理、财务诊断、投资诊断、职业判断、心理诊断或医疗诊断。
- 初始扫描、生态观察、能量观察、钱包汇总和收入来源入账都由确定性逻辑完成。

收入来源必须保持手动和非审判式：

- 用户手动维护收入来源。
- 用户手动确认每一次入账。
- 频率只用于展示，不触发自动到账、提醒或预测。
- 系统不判断收入稳定性。
- UI 不告诉用户应该怎么赚钱、花钱、存钱或投资。
- UI 不暗示连接了银行、支付平台、雇主系统、税务系统或外部账户。
- UI 不用“警报”“危险”“你应该”“你必须”等训诫文案。

界面语言使用简体中文。

推荐顺序：

1. 先完成 `income-source-local-data.md`。
2. `income-source-panel.md` 可以在类型稳定后开始。
3. `income-source-manual-deposit.md` 依赖收入来源类型和钱包保存函数，可以与面板任务部分并行，但最终需要集成到页面。
